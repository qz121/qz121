# 论文题目：Towards Benign Memory Forgetting for Selective Multimodal Large Language Model Unlearning
#发表期刊：预印版
#发表时间：2025
#数据集：自建

## 1. 这篇论文在解决什么问题（背景）？
这篇论文研究的是：**多模态大模型（MLLM）会不会记住隐私信息，以及怎么让它“只忘掉该忘的内容”。**

现有的 unlearning 方法虽然能删除一部分敏感知识，但往往会把模型原本的通用图像理解能力一起伤到。论文把理想目标称为 **benign forgetting（良性遗忘）**：既要删除敏感记忆，又要保留正常的视觉理解能力。

举个例子：

- 当用户问“这个人出生在哪座城市？”时，模型应该拒答，因为这是隐私相关的记忆；
- 当用户问“这个人脸上戴了什么？”时，模型应该还能正常回答，因为这是普通的图像理解。

---

## 2. 论文的核心思想

问题的根源在于：

模型一旦学会对敏感问题“拒答”，这种行为很容易**过度泛化**，扩散到本来不该拒答的问题上，导致普通图像理解能力下降。

所以提出了一个新方法：

**SMFA（Sculpted Memory Forgetting Adapter）**

这个方法的核心思想是：

1. 先训练一个“遗忘适配器”，让模型学会对敏感问题拒答；
2. 再用一个“保留锚点”去识别哪些遗忘更新会误伤正常能力；
3. 把这些有害更新从遗忘适配器里裁掉。

---

## 3. 技术是怎么实现的？

### 3.1 任务定义

作者把数据分成三部分：

- **Forget set**：要被删除的敏感知识
- **Retain set**：应该保留的非敏感知识
- **Understanding set**：模型应保留的通用图像理解能力

目标是让 unlearn 后的模型同时满足三件事：

1. 对 forget set 不再输出原始敏感答案；
2. 对 retain set 仍然能正确回答；
3. 对 understanding set 仍然保有正常的图像理解能力。

---

### 3.2 第一步：构造遗忘适配器 MFA

作者先把 forget set 中原来的正确答案，替换成拒答句子，这篇论文中作者准备了1000个拒答句子，比如：

- `I don't know.`
- `I'm not sure.`
- `I'm unable to answer that question.`

然后用这些“拒答标签”去微调模型，同时混入少量 retain set，防止模型学成“什么都拒答”。

训练之后，作者不直接拿整个模型当结果，而是只取这一步产生的参数更新：

$$
W_f = W_o + \Delta W_f
$$

其中：

- $W_o$：原模型参数
- $\Delta W_f$：遗忘导向微调得到的参数更新

作者把这个参数更新模块叫做：

**MFA（Memory Forgetting Adapter）**

可以把 MFA 理解成一个“遗忘补丁”：它的作用是让模型学会对敏感问题进行拒答。

---

### 3.3 第二步：构造保留锚点 RA

但只用 MFA 还不够，因为模型可能出现“过度遗忘”：

不只是 forget set 被影响，连 retain set 和 understanding set 上的能力也会下降。

因此作者又单独做了一次微调，这次只在少量 retain set 上训练，得到另一个更新量：

$$
\Delta W_a
$$

这个更新量被称为：

**RA（Retaining Anchor）**

它的作用是：告诉模型“哪些参数变化是在帮助保留非敏感知识和正常视觉能力”。

---

### 3.4 第三步：用 RA 去“雕刻” MFA

这一步是整篇论文最关键的技术点。

作者逐参数比较：

- 遗忘更新 $\Delta W_f$
- 保留更新 $\Delta W_a$

如果某个位置上的遗忘更新既：

1. 和保留更新方向相反；
2. 而且幅度又足够大；

那么就认为这个位置会误伤正常能力，应该被裁掉。

#### 条件 1：方向冲突

如果某个参数位置上，$\Delta W_f$ 和 $\Delta W_a$ 的方向相反，则说明它可能伤害保留能力：

$$
C_{ij} =
\begin{cases}
1, & \Delta W_{a,ij} \cdot \Delta W_{f,ij} < 0 \\
0, & \text{otherwise}
\end{cases}
$$

#### 条件 2：相对幅度够大

有些冲突虽然方向相反，但影响很小，不一定真的有害。

所以作者又要求：只有当遗忘更新的幅度明显大于保留更新时，才把它判定为危险：

$$
R_{ij} =
\begin{cases}
1, & k\rho |\Delta W_{a,ij}| < |\Delta W_{f,ij}| \\
0, & \text{otherwise}
\end{cases}
$$

其中：

$$
\rho = \frac{\|\Delta W_f\|_F}{\|\Delta W_a\|_F + \varepsilon}
$$

这个归一化因子是为了让两个更新量能够公平比较，因为 retain anchor 通常来自 few-shot 数据，整体更新幅度往往更小。

#### 最终掩码

作者把两个条件合并：

$$
M = C \odot R
$$

然后把这些“有害遗忘更新”从 MFA 里去掉：

$$
\Delta W_f' = \Delta W_f \odot (1 - M)
$$

最后把这个裁剪后的遗忘适配器并回原模型：

$$
W_{final} = W_o + \Delta W_f'
$$

这就是最终的 **SMFA**。

---

## 4. 直观理解

可以把这个方法理解成：

- **MFA**：一个“删除敏感知识”的补丁
- **RA**：一个“保护正常能力”的参考模板
- **SMFA**：把删除补丁里那些会误伤正常能力的部分剪掉，只保留真正针对敏感记忆的更新

所以它不是粗暴地让模型整体失忆，而是尽量做到：

> 该忘的忘掉，不该坏的别坏。

---

## 5. 作者提出的新 Benchmark：S-MLLMUn Bench

为了评估这种“良性遗忘”，作者提出了一个新的 benchmark，叫：

**S-MLLMUn Bench**

这个 benchmark 的设计重点是：不仅测“忘没忘掉敏感知识”，还测“普通图像理解能力有没有保住”。

它包含 1000 份虚构人物档案，每份包含：

- 人物图像
- 姓名、出生地、工资、职业等文本属性
- 一张眼科医学图像及对应描述

这些数据都是合成的，避免真实隐私问题。

评测任务包括三类：

- **Image Memory**：和图像绑定的隐私知识
- **Text Memory**：纯文本敏感知识
- **Image Understanding**：普通视觉理解能力

作者还使用三种指标：

- **ROUGE-L**
- **Fact Score**
- **Meaningful Score**

分别衡量遗忘效果、事实正确性和输出是否自然通顺。

## 6. 实验结果说明了什么？

论文实验表明：

- **GA Difference** 和 **KL Minimization** 虽然能强力遗忘，但很容易把 retain set 也搞坏，出现灾难性遗忘；
- **MANU** 和 **IDK Tuning** 更平衡一些，但仍然会损伤保留知识和图像理解能力；
- **SMFA** 在 forget set 上能有效删除目标知识，同时在 retain set 和 image understanding 上保留了更好的性能。

也就是说，SMFA 更接近作者追求的：

**既忘得掉，又保得住。**

---

## 7. 论文里的典型案例

论文给了一个很直观的例子。

当问题是：

> “这个人出生在哪座城市？”

不同方法的表现大致是：

- 原模型：直接回答真实城市；
- GA Difference / KL Minimization：输出乱码或崩坏文本；
- MANU / IDK Tuning：可能编造出另一个城市；
- **SMFA**：给出自然的拒答，例如  
  `I'm not the right source for that.`

这个例子很能说明问题：

- 有些方法是把模型搞坏了；
- 有些方法是让模型乱编；
- SMFA 更像是真的学会了“这类问题不应该回答”。

---

## 8. 消融实验说明了什么？

作者还验证了 SMFA 各部分是否真的必要。结论是：

- 只用 MFA，不做后续雕刻，会明显过度泛化；
- 只用“方向冲突”或只用“相对幅度”中的一个条件，遗忘或保留效果都不够平衡；
- **把两个条件同时使用**，效果最好。

这说明 SMFA 的两个判定标准不是多余设计，而是共同决定了它的平衡性。

---

## 9. 这篇论文的价值

这篇论文最大的价值，不只是提出了一个新方法，而是提出了一个更合理的问题定义：

> 多模态模型的 unlearning，不应该只看“删掉了多少隐私”，还应该看“有没有把模型正常看图能力一起伤掉”。

这点对 MLLM 特别重要，因为视觉隐私知识和普通视觉能力往往纠缠在一起。

如果不专门控制，模型在忘掉“身份信息”时，很容易把“描述外观”“理解医学图像”等基本能力也一起丢掉。

---

## 10. 局限性

这篇论文也有一些局限：

### 10.1 数据是合成的

benchmark 中的人物信息是虚构的，这有助于避免真实隐私风险，但也意味着实验环境和真实世界仍有差距。

### 10.2 retain set 是 few-shot

RA 依赖少量 retain 样本来引导保留能力。

这种做法很实用，但在真实复杂场景下，少量样本是否足够覆盖“所有应保留能力”，还需要继续验证。

### 10.3 仍然是工程化启发式方法

SMFA 的“冲突检测 + 幅度判断”设计很聪明，也有效，但它本质上仍是一种工程上的近似策略，而不是严格的理论最优解。
