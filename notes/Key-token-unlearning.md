# 论文笔记：Visual-Guided Key-Token Regularization for Multimodal Large Language Model Unlearning
### 论文链接：https://arxiv.org/pdf/2601.22020  
### 数据集：MLLMU、CLEAR  
## 1. 背景

这篇论文研究的是 **多模态大语言模型（MLLM）的遗忘 / unlearning** 问题：当模型记住了某些图像中的私人信息时，如何让它在被提问时**不再泄露这些信息**，同时又尽量保留其他知识和回答连贯性。论文指出，已有方法往往把回答中的所有 token 一视同仁，容易把本来不该忘的“正常 token”也一起破坏掉，导致输出变得不连贯甚至乱码。

论文提出的方法叫 **ViKeR（Visual-Guided Key-Token Regularization）**。核心思想是：先借助**无关参考图像**估计“理想的遗忘后 token 分布”，再用这个分布去约束真正的遗忘训练，从而更有针对性地遗忘**关键 token**，尽量保住**正常 token**。

---

## 2. 论文要解决的核心问题

传统遗忘方法的主要问题有两个：

1. **token 一视同仁**  
   句子里的所有 token 都被同等对待，结果容易把原本只是负责句子结构的 normal tokens 也忘掉。

2. **只看语言，不看视觉**  
   以前的方法大多只从文本角度处理遗忘，忽略了图像本身可以帮助判断哪些 token 更关键。论文认为，视觉输入能帮助区分：
   - **normal tokens**：维持句子结构、但不泄露隐私
   - **key tokens**：直接携带敏感/身份信息的 token 

---

## 3. 技术路线总览

ViKeR 的技术路线可以概括成两步：

### Step 1：视觉引导的 token 分布估计
用一些**和 forget set 无关的参考图像**，把同一个问题输入到未遗忘前的完整模型中，得到每个 token 位置上的概率分布。然后把多个参考图像对应的分布**逐词表维度求平均**，得到一个“理想的遗忘后分布”。

### Step 2：token-level regularization
在真正遗忘时，让当前模型的输出分布尽量接近上一步估计出来的理想分布。论文使用 **KL divergence** 作为正则项，把“该留的 token”稳住，把“该忘的 token”压下去。

---

## 4. ViKeR 的具体工作流程

### 4.1 输入数据
论文只需要两类东西：

- **forget set**：需要被遗忘的图像-问题-答案三元组
- **reference set**：k个与 forget set 无关的参考图像，用来估计理想分布

实验里一般取 `k = 5` 张参考图像。

### 4.2 先估计“理想分布”
对同一个forget set问题 `x`，把 `k` 张参考图像分别送进完整模型 `θ_full`，得到 `k` 个 token 分布：

$$p_{\theta_{full}}(v \mid l'_j, x, i)$$

然后对同一位置 `i`、同一词 `v` 的概率做平均：

$$\hat R_i^y(v) = \frac{1}{k}\sum_{j=1}^k p_{\theta_{full}}(v \mid l'_j, x, i)$$

这一步的意思是：**每张参考图都会给出一个“这个位置上应该怎么说”的概率分布，把它们平均起来，就得到一个更稳定的参考分布。**

### 4.3 再进行真正的遗忘训练
当前模型 `θ` 在 forget set 上进行遗忘训练时，不是单纯把答案整体打坏，而是优化下面这个目标：

$$\
L_{ViKeR}(D_f;\theta) = -L_{NLL}(D_f;\theta) + \lambda \sum KL(\hat R_i^y \,\|\, \hat Q_i^y)
\$$

其中：

- `-L_NLL`：相当于 GA 风格的遗忘项
- `KL(...)`：正则项，让当前模型输出靠近参考分布
- `λ`：平衡系数

---

## 5. 关键概念解释

### 5.1 什么是 token-level？
这里的 token-level 指的是：**把回答拆成一个个 token，分别看每个 token 对训练目标的贡献**。  
论文把 GA 的梯度写成 token 级求和形式，说明遗忘不是对整句一次性处理，而是对每个 token 都有单独的梯度信号。

直观上，比如答案是：

> `This person was born in Copenhagen`

那么 token-level 就是分别看：
- `This`
- `person`
- `was`
- `born`
- `in`
- `Copenhagen`

每个 token 都会单独影响梯度。

---

### 5.2 GA 的 token-level 是什么？
GA（Gradient Ascent）是最基础的遗忘方法之一。它的做法是：  
**对 forget set 的正确答案做“反向学习”**，让模型不要再倾向于输出这些答案。

token-level 版本表示：GA 不是只对整个句子给一个总梯度，而是对每个 token 都有一个梯度贡献。论文指出，GA 会把所有 token 都一起往反方向推，结果可能把 normal tokens 也误伤掉。

---


## 6. “k 个分布按词表维度逐项求平均”是什么意思

假设词表里只有 4 个词：`A, B, C, D`。

三张参考图分别输出的分布是：

- 图 1：`[0.1, 0.2, 0.6, 0.1]`
- 图 2：`[0.2, 0.1, 0.5, 0.2]`
- 图 3：`[0.0, 0.3, 0.5, 0.2]`

那么逐项求平均就是：  
（0.1+0.2+0.0）÷3  
（0.2+0.1+0.3）÷3  
（0.6+0.5+0.5）÷3  
（0.1+0.2+0.2）÷3  
意思是：**对同一个 token 位置，多个参考图像都会给出一个词表概率分布，把它们在每个词上的概率平均起来，就得到最终参考分布。**

---

## 7. KL 散度怎么理解

KL 散度衡量的是两个分布之间的差异：

\$$
KL(R\|Q) = \sum_v R(v)\log\frac{R(v)}{Q(v)}
\$$

在 ViKeR 里：

- `R`：参考图像平均得到的“理想分布”
- `Q`：当前模型在 forget set 上的输出分布

训练时最小化 KL，就是让当前模型的输出尽量接近参考分布。

### 例子
假设某个位置上只有 4 个候选词：

- `R = [0.7, 0.2, 0.05, 0.05]`
- `Q = [0.4, 0.2, 0.2, 0.2]`

那么：

$$
\begin{aligned}
D_{KL}(R \parallel Q) &= 0.7 \ln \frac{0.7}{0.4} + 0.2 \ln \frac{0.2}{0.2} + 0.05 \ln \frac{0.05}{0.2} + 0.05 \ln \frac{0.05}{0.2} \\
&= 0.7(0.5596) + 0.2(0) + 0.05(-1.3863) + 0.05(-1.3863) \\
&\approx 0.3917 + 0 - 0.0693 - 0.0693 \\
&\approx 0.253
\end{aligned}
$$

这表示当前分布 `Q` 和参考分布 `R` 还有明显差距。  
在 ViKeR 里，优化这个 KL 的目标不是恢复原答案，而是让模型保持**合理的 token 结构**，不要把句子忘坏。

---

## 8. 为什么论文要用 GA，而不是直接用 PO / DPO / NPO？

论文这一版 ViKeR 是建立在 **GA 的 token-level 梯度分析** 上的，所以它的主干就是 GA + 正则项。  
论文并不是说 PO / DPO / NPO 不可行，而是这篇工作本身选择了 GA 作为基础，因为这样更容易从梯度层面分析“哪些 token 被忘掉、哪些 token 被保住”。

换句话说：

- **GA**：适合做 token 级梯度重加权分析
- **PO / DPO / NPO**：是别的遗忘思路，但不是这篇论文的主设计
- **ViKeR**：本质上是一个“带视觉引导的正则化遗忘目标”

---

## 9. 实验结论

论文在 **MLLMU** 和 **CLEAR** 数据集上验证了 ViKeR。结果显示：

- 它能达到较好的遗忘效果
- 同时比 GA、NPO、IdkPO 更能保持回答连贯性
- 对非目标知识的保留更好
- 输出更不容易变成乱码或拒答过度
